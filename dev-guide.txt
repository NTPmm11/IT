วิธีอ่านไฟล์นี้ง่ายมาก: ไล่จากบนลงล่าง ทีละขั้น อย่าข้าม
ไม่ต้องรีบ ไม่ต้องจำทุกอย่าง ทำตามไปเรื่อยๆ เดี๋ยวเป็นเอง
พังก็ไม่เป็นไร เพราะทุกอย่างย้อนกลับได้ (เดี๋ยวสอนวิธี)



🏁 ขั้นที่ 0 — เตรียมเครื่อง (ทำครั้งเดียวจบ)
──────────────────────────────────────────────

ต้องมี 3 อย่าง:

  1. Browser — Chrome หรือ Edge (มีอยู่แล้วแหละ)
  2. VSCode — โปรแกรมแก้โค้ด ฟรี
     โหลดที่ code.visualstudio.com
     ลงเสร็จเปิดขึ้นมา -> File -> Open Folder
     -> เลือกโฟลเดอร์โปรเจคนี้ (IT)
  3. Node.js — ตัวรันคำสั่ง npm (ไว้เปิดโปรเจค)
     โหลดที่ nodejs.org เอาตัว LTS แล้วกด next รัวๆ ได้เลย
     ลงเสร็จเช็คหน่อยว่ามาจริง: เปิด Terminal พิมพ์  node -v
     ขึ้นเลข version = เรียบร้อย

แค่นี้พอ เริ่มได้เลย

ของแถม 1 ชิ้น (ไม่ลงก็ได้ แต่ลงแล้วชีวิตดีขึ้นเยอะ):

  • Vue.js devtools — extension ของ Chrome/Edge
    ค้นใน Chrome Web Store ว่า "Vue.js devtools"
    ได้อะไร: เปิด F12 แล้วจะมีแท็บ Vue โผล่มา
    เห็นค่าตัวแปรในหน้าเว็บเปลี่ยนสดๆ ตอนเราพิมพ์/กดปุ่ม
    ตอนเรียน Vue (ท้ายไฟล์นี้) จะรักมันมาก
    (โบนัส: ตอนรัน npm run dev เทอร์มินัลจะโชว์ลิงก์
     http://localhost:xxxx/__devtools__/ ด้วย — เป็น Vue devtools
     แบบในตัว ไม่ต้องลง extension ก็ใช้ได้เหมือนกัน)

  ⚠ ไม่ต้องลง Live Server แล้วนะ — โปรเจคนี้เปลี่ยนมาใช้ Vite
  (ตัวรัน dev server ของ Vue) ซึ่งมี hot-reload ในตัวอยู่แล้ว
  แก้โค้ดแล้ว save ปุ๊บ browser เปลี่ยนให้เองทันที ไม่ต้องพึ่ง
  extension ไหนเสริมอีก



🏠 ขั้นที่ 1 — เว็บเพจทำงานยังไง (อ่านแป๊บเดียว)
──────────────────────────────────────────────

เว็บ 1 หน้า = ของ 3 ชนิดช่วยกันทำงาน นึกภาพเป็นบ้าน:

  HTML  = โครงบ้าน    มีห้องอะไร ประตูอยู่ไหน
                       (ปุ่ม ช่องกรอก ตาราง วางตรงไหนของหน้า)
  CSS   = สีทาบ้าน    สวยยังไง สีอะไร ใหญ่เล็กแค่ไหน
  JS    = ไฟฟ้าประปา  กดสวิตช์แล้วเกิดอะไรขึ้น
                       (กดปุ่มนี้แล้วให้ทำอะไร)

browser อ่านของพวกนี้แล้ววาดหน้าเว็บให้เราดู จบ

ในโปรเจคนี้ 3 ชนิดนี้ "รวมอยู่ในไฟล์เดียว" ต่อหนึ่งหน้า
เรียกไฟล์แบบนี้ว่า .vue (Single File Component) แบ่งเป็น 3 โซนชัดเจน:

  <template>  ...  </template>   ← โครง (เหมือน HTML เดิม)
  <script>    ...  </script>     ← การทำงาน (เหมือน JS เดิม)
  <style>     ...  </style>      ← หน้าตา (ลิงก์ไป CSS ที่ใช้ร่วม)

เปิด frontend/src/views/LoginView.vue ดูเป็นตัวอย่างได้เลย
สามโซนอยู่ไฟล์เดียว ไม่ต้องสลับไฟล์ไปมาเหมือนเมื่อก่อน



👀 ขั้นที่ 2 — เปิดโปรเจคดูของจริงกันเลย
──────────────────────────────────────────────

ตอนนี้โปรเจคแยกเป็น 2 ฝั่งแล้ว:
  หน้าบ้าน (frontend) = หน้าเว็บที่เราเห็น
  หลังบ้าน (backend)  = server ที่รับข้อมูลไปเก็บลง database

  1. เปิด VSCode ที่โฟลเดอร์ IT
  2. เปิด Terminal ในตัว VSCode (เมนูบน: Terminal -> New Terminal)
  3. เปิดหน้าบ้านก่อน พิมพ์:

       cd frontend
       npm install      (ครั้งแรกครั้งเดียว โหลด Vue/Vite มาเก็บในเครื่อง)
       npm run dev

     รอแป๊บนึง เทอร์มินัลจะบอก URL เช่น http://localhost:5173/
     (ถ้าพอร์ตชนกับโปรแกรมอื่น Vite จะขยับไปพอร์ตถัดไปเอง
      เช่น 5174 — ดูเลขที่เทอร์มินัลบอกจริงเป็นหลัก)
     เปิด browser ไปที่ URL นั้น
     (อยากปิด server เมื่อไหร่ กลับมากด Ctrl+C ใน Terminal)
  4. หน้า login ตอนนี้เช็ครหัสกับ database จริงแล้ว
     ต้องเปิดหลังบ้าน + ตั้งค่า database ก่อนถึงจะ login ผ่าน
     -> วิธีทำอยู่ "ภาคพิเศษ 2: หลังบ้าน" ท้ายไฟล์นี้
     (ยังไม่ตั้งก็เปิดดูหน้าตาเว็บได้ แค่ login ไม่ผ่านเฉยๆ)
  5. เข้ามาหน้าฟอร์ม CR แล้ว -> ลองกรอกเล่น
     ลองกดปุ่ม "+ เพิ่มขั้นตอนงาน" ดู
  6. กด Submit -> ข้อมูลลง database แล้วเด้งไปหน้าอนุมัติ
     (เด้งแบบไม่โหลดหน้าใหม่นะ — เพราะเป็น Vue Router
      สลับหน้าโดยไม่ reload browser เลย เร็วกว่าเดิมด้วย)

หมายเหตุนิดนึง: ต้องต่อเน็ตตอนเปิดนะ เพราะไอคอน (Font Awesome)
ยังโหลดมาจากเน็ตอยู่ (ส่วน Vue เองตอนนี้ติดตั้งอยู่ในเครื่องแล้ว
ผ่าน npm install ไม่ต้องพึ่งเน็ตแล้ว)



✏️  ขั้นที่ 3 — แก้โค้ดครั้งแรกในชีวิต
──────────────────────────────────────────────

ทำตามนี้เป๊ะๆ ใช้เวลาไม่ถึง 10 นาที:

  1. เปิด VSCode ที่โฟลเดอร์โปรเจค
  2. เปิดไฟล์ frontend/src/views/LoginView.vue
  3. หาบรรทัดที่เขียนว่า   <h1>Login</h1>   (อยู่ในโซน <template>)
  4. แก้เป็น              <h1>Login ระบบ CR</h1>
  5. กด save (Ctrl+S)
  6. กลับไปดู browser เลย — ไม่ต้องกด refresh เอง
     Vite มี Hot Module Reload ในตัว เปลี่ยนให้อัตโนมัติ
  7. ข้อความบนจอเปลี่ยน!

ยินดีด้วย เขียนโค้ดเป็นแล้ว 1 บรรทัด

จำ loop นี้ไว้ให้ขึ้นใจ เพราะจะทำแบบนี้ไปตลอด:

     แก้ -> save -> ดูผลใน browser (เปลี่ยนให้เองอัตโนมัติ) -> แก้ต่อ -> ...



🗂  ขั้นที่ 4 — อะไรอยู่ไฟล์ไหน
──────────────────────────────────────────────

  IT/
  ├─ frontend/                ← หน้าบ้าน (หน้าเว็บทั้งหมด, Vite + Vue Router)
  │   ├─ index.html             จุดเข้าเว็บของ Vite — แทบไม่ต้องแตะ
  │   └─ src/
  │       ├─ main.js            สั่ง Vue เริ่มทำงาน + ผูก router (ทำครั้งเดียว)
  │       ├─ App.vue             เปลือกนอกสุด มีแค่ <router-view/>
  │       ├─ router/
  │       │   └─ index.js        กำหนดว่า path ไหนโชว์หน้าไหน
  │       │                      ( / -> login, /form -> ฟอร์ม, /approve -> อนุมัติ)
  │       ├─ views/             ← 1 ไฟล์ = 1 หน้า (โครง+การทำงานรวมกัน)
  │       │   ├─ LoginView.vue    หน้า login
  │       │   ├─ FormView.vue     หน้าฟอร์ม CR
  │       │   └─ ApproveView.vue  หน้าอนุมัติ
  │       ├─ services/          ← ของที่หลายหน้าใช้ร่วมกัน
  │       │   ├─ api.js           ที่อยู่ API + ตัวช่วยยิง API (apiFetch)
  │       │   └─ commonActions.js ปุ่มที่ใช้ร่วมกันทุกหน้า (ยกเลิก/ร่าง/PDF)
  │       └─ assets/
  │           ├─ css/            ← สีทาบ้าน (หน้าตา)
  │           │   ├─ base.css      ของที่ใช้ทุกหน้า (พื้นหลัง ปุ่ม)
  │           │   ├─ login.css     เฉพาะหน้า login
  │           │   └─ form.css      หน้าฟอร์ม + หน้าอนุมัติ
  │           └─ img/            รูปภาพ
  ├─ backend/                  ← หลังบ้าน (server — ดูภาคพิเศษ 2)
  │   ├─ .env                    รหัสผ่าน database ของเครื่องเรา (ห้าม commit)
  │   ├─ scripts/                เครื่องมือช่วย เช่น สร้าง hash รหัสผ่าน
  │   └─ src/
  │       ├─ index.js            จุดสตาร์ท server (port 4000)
  │       ├─ db.js               ตัวต่อกับ SQL Server
  │       ├─ middleware/         ด่านเช็ค token ก่อนเข้า API
  │       └─ routes/             API แต่ละเส้น (auth / systems / cr)
  ├─ database/
  │   └─ schema.sql            พิมพ์เขียว database (ตาราง 6 ตัว)
  └─ README.md                 แผนที่โปรเจคฉบับย่อ

กติกาบ้านนี้มีแค่ 3 ข้อ (ยังเหมือนเดิม แค่ย้ายที่อยู่):

  1. โครง/การทำงาน/หน้าตา — แยกโซนชัดใน .vue (<template>/<script>/<style>)
  2. ของที่หลายหน้าใช้ร่วมกัน -> ไปอยู่ assets/css/base.css / services/commonActions.js
  3. ของเฉพาะหน้าไหน -> ไปอยู่ในไฟล์ view ของหน้านั้น

ตารางลัด "อยากแก้ตรงนี้ ต้องไปไฟล์ไหน":

  เปลี่ยนสีปุ่มทุกหน้า          -> frontend/src/assets/css/base.css
  หน้าตาฟอร์ม/ตาราง             -> frontend/src/assets/css/form.css
  เพิ่มช่องกรอกในฟอร์ม          -> frontend/src/views/FormView.vue
                                   (<template> เพิ่มช่อง, <script> เพิ่มตัวแปรใน data()
                                   + เพิ่ม column ใน database ถ้าจะเก็บจริง)
  ข้อความปุ่มยกเลิก/ร่าง/PDF    -> frontend/src/services/commonActions.js
  เพิ่ม/แก้ user ที่ login ได้   -> ตาราง users ใน database
                                   (ไม่ได้อยู่ใน LoginView.vue แล้ว)
  เพิ่มตัวเลือก dropdown ระบบ    -> ตาราง systems ใน database
  เพิ่มหน้าใหม่                  -> สร้างไฟล์ใน src/views/ แล้วเพิ่ม route
                                   ใน src/router/index.js
  เพิ่ม API เส้นใหม่             -> backend/src/routes/

บอกไว้ก่อน: ทุกไฟล์มี comment ภาษาไทยอธิบายเกือบทุกบรรทัดแล้ว
เปิดไปอ่านได้เลย ไม่ต้องกลัวอ่านไม่รู้เรื่อง
(comment = ข้อความหลัง // หรือใน <!-- --> ที่ browser ไม่สนใจ
 เขียนไว้ให้คนอ่านเข้าใจโค้ดเฉยๆ)



📚 ขั้นที่ 5 — เรียนอะไรก่อนหลัง (ห้ามข้ามขั้นนะ)
──────────────────────────────────────────────

【 1 】 HTML
   เรียนแค่: แท็กพื้นฐาน, form, input, table, label
   เรียนที่: w3schools.com/html — มีปุ่ม "Try it Yourself"
             ลองโค้ดในเว็บได้เลย ไม่ต้องลงอะไร
   หรือ YouTube ค้น "HTML พื้นฐาน" สอนไทยเพียบ
   โยงกับโปรเจค: เรียนแท็กไหนจบ เปิด src/views/FormView.vue
     ดูโซน <template> หาแท็กนั้นดู ของจริงอยู่ครบ
     <form> <input> <select> <table> <label>
     เช่นเรียน table จบ ไปดูตาราง action plan ว่าเขียนยังไง

【 2 】 CSS
   เรียนแค่: selector, class, สี, ขนาด, flexbox, grid
   เรียนที่: w3schools.com/css
   โยงกับโปรเจค: เปิด src/assets/css/base.css ลองเปลี่ยนสีปุ่มเล่น
     เรียน class จบ ไปดูว่าปุ่มในโปรเจคใช้ class อะไรบ้าง
     เรียน flexbox/grid จบ ไปส่องใน src/assets/css/form.css
     ว่าฟอร์มจัดช่องกรอกเรียงกันสวยๆ ได้ยังไง

【 3 】 JavaScript — ตัวนี้เนื้อเยอะสุด ค่อยๆ ไป ไม่ต้องรีบ
   เรียนแค่: ตัวแปร, if/else, function, array, object
   เรียนที่: w3schools.com/js หรือ javascript.info
   รู้แค่ 5 เรื่องนี้ก็อ่านโปรเจคนี้ออกแล้ว ไม่ต้องเรียนหมด
   โยงกับโปรเจค: ทั้ง 5 เรื่องมีของจริงให้ดูในโปรเจคหมดเลย
     if/else + function  -> src/views/LoginView.vue (โซน <script>, เช็ครหัสผ่าน)
     array + object      -> src/views/FormView.vue (ตัวแปร rows คือ
                            array ของ object = แถวในตาราง)
     เรียนเรื่องไหนจบ เปิดไฟล์นั้นแล้วหาให้เจอว่าอยู่ตรงไหน

【 4 】 Vue 3
   อย่างที่รู้กันว่าโปรเจคนี้ใช้ Vue (+ Vue Router ให้สลับหน้า)
   หน้าที่มัน: ทำให้ HTML กับ JS คุยกันง่ายๆ
   รู้ 6 คำนี้พอ:
     data()    = ตัวแปรของหน้า
     v-model   = ผูกช่องกรอกกับตัวแปร (พิมพ์ปุ๊บค่าเปลี่ยนปั๊บ)
     v-for     = วนสร้างของซ้ำๆ จากรายการ (ตารางใช้ตัวนี้)
     @click    = กดปุ่มแล้วเรียก function
     methods   = function ของหน้า
     route     = เส้นทาง URL ไหน โชว์หน้า (component) ไหน
   ★ เรื่อง Vue มีบทพิเศษเขียนละเอียดไว้ท้ายไฟล์นี้
     ชื่อ "ภาคพิเศษ: เรียน Vue กับโปรเจคนี้" — เลื่อนลงไปดู

【 5 】 เครื่องมือคู่กาย
   DevTools: กด F12 ใน browser -> แท็บ Console
     - อะไรพังจะขึ้นสีแดง บอกไฟล์+บรรทัดที่พังเสมอ
     - แท็บ Network ข้างๆ ก็เด็ด: กด Submit แล้วจะเห็น
       request ที่หน้าเว็บยิงไปหา backend พร้อมข้อมูลข้างใน
   Git: สมุดบันทึก version ของโค้ด พังแล้วย้อนกลับได้
     เรียน 3 คำสั่งพอ: git status / git add / git commit
     เรียนที่: YouTube ค้น "git พื้นฐาน"
     โยงกับโปรเจค: โปรเจคนี้เป็น git repo อยู่แล้ว
     ฝึกได้เลย — ทำโจทย์ขั้นที่ 6 เสร็จข้อไหน commit เก็บทีนึง



💪 ขั้นที่ 6 — โจทย์ฝึกมือ (เรียงจากง่ายไปยาก)
──────────────────────────────────────────────

โจทย์ 1: เปลี่ยนสีปุ่ม Submit
  ไปที่ src/assets/css/base.css หา .btn-submit แก้ค่า background
  ลองใส่ #e91e63 ดู ชมพูจี๊ดเลย

โจทย์ 2: เพิ่มตัวเลือกใน dropdown "ระบบที่เกี่ยวข้อง"

  แล้วในโซน <script> เพิ่มตัวแปรใน data() ให้มัน

โจทย์ 4: เพิ่มปุ่ม "ล้างฟอร์ม"
  เพิ่ม method ใหม่ในโซน <script> ของ FormView.vue
  แล้วเพิ่มปุ่มในโซน <template> ผูกกันด้วย @click

▶ Vue ตอนที่ 1: Vue คือะไร ทำไมต้องมี
──────────────────────────────────────────────

ลองนึกภาพ: อยากให้เลข "ลำดับแถว" ในตารางอัปเดตทุกครั้ง
ที่มีคนเพิ่มหรือลบแถว

ถ้าใช้ JS เปล่าๆ ต้องสั่งเองหมดเลยนะ:
  - สร้างแถวใหม่เอง (document.createElement บลาๆ)
  - หาตำแหน่งแทรกเอง
  - ไล่แก้เลขลำดับของทุกแถวเองอีกรอบ
  ลืมขั้นไหนสักขั้น = จอโชว์ข้อมูลผิดทันที

Vue บอกว่า: ไม่ต้องทำขนาดนั้น คิดใหม่แบบนี้

  "เราดูแลแค่ ข้อมูล (ตัวแปร) — หน้าจอเดี๋ยว Vue วาดตามให้เอง"

นึกถึง Excel ก็ได้: ใส่สูตร =A1+A2 ไว้ครั้งเดียว
พอแก้ค่าใน A1 ช่องผลรวมก็เปลี่ยนเอง ไม่ต้องกดอะไรซ้ำ
Vue คือแบบนั้นเลย ผูกตัวแปรกับจอไว้ครั้งเดียว
แก้ตัวแปรปุ๊บ จอเปลี่ยนปั๊บ

แล้วต้องติดตั้งอะไรไหม? ตอนนี้ติดตั้งไว้ให้แล้ว
(ทำตอน scaffold โปรเจคด้วย npm create vue@latest — เก็บ Vue ไว้ใน
frontend/node_modules) เราแค่ import ใช้งานในไฟล์ .vue ได้เลย
ไม่ต้องพึ่งเน็ตโหลด Vue จาก CDN เหมือนเมื่อก่อนแล้ว


▶ Vue ตอนที่ 2: 6 คำหลัก + โค้ดจริงจากโปรเจคเรา
──────────────────────────────────────────────

เปิด src/views/LoginView.vue กับ src/views/FormView.vue ตามไปด้วยนะ
เห็นภาพกว่าเยอะ (ทั้ง template และ script อยู่ไฟล์เดียวกันแล้ว)

── คำที่ 1: route + view = จุดสตาร์ทของแต่ละหน้า ──

  ทุกหน้าเคยต้องเขียน createApp({...}).mount("#app") เอง
  ตอนนี้ทำครั้งเดียวจบใน src/main.js + src/App.vue แทน
  หน้าที่เราแค่เขียน "view component" ธรรมดา 1 ไฟล์ต่อ 1 หน้า
  แล้วบอก Vue Router ว่า path ไหนใช้ component ไหน — อยู่ใน
  src/router/index.js:

    { path: '/',       component: LoginView }
    { path: '/form',    component: FormView }
    { path: '/approve', component: ApproveView }

  พอ URL เปลี่ยน (หรือกด this.$router.push("/form")) Vue Router
  จะสลับ component ให้เอง โดยไม่ reload หน้าเว็บทั้งหน้า

── คำที่ 2: data() = ตัวแปรของหน้า ──

  data() {
    return {
      username: "",     // ค่าเริ่มต้น = ว่าง
      password: ""
    };
  }

(นี่จาก src/views/LoginView.vue) ตัวแปรใน data() ไม่ใช่ตัวแปรธรรมดานะ
Vue "เฝ้าดู" มันตลอดเวลา ค่าเปลี่ยนเมื่อไหร่
จอส่วนที่ผูกไว้เปลี่ยนตามทันที — นี่แหละหัวใจของ Vue

── คำที่ 3: v-model = สายเชื่อมช่องกรอกกับตัวแปร ──

  <input v-model="username">

พิมพ์ในช่อง -> ตัวแปร username เปลี่ยนตาม (เอง)
โค้ดแก้ตัวแปร -> ข้อความในช่องก็เปลี่ยนตาม (เอง)
เชื่อมสองทางแบบนี้เรียกว่า two-way binding

ในโปรเจคเราใช้ v-model ครบ 4 ท่า (ดูได้ในโซน <template>
ของ src/views/FormView.vue):

  ท่า 1: ช่องข้อความ -> ค่าเป็นข้อความ
     <input v-model="form.crId">
  ท่า 2: radio -> เลือกอันไหน ค่าเป็นอันนั้น
     <input type="radio" value="Low" v-model="form.priority">
  ท่า 3: checkbox หลายอัน + ตัวแปรเป็น array
     <input type="checkbox" value="App" v-model="form.changeTypes">
     ติ๊ก 2 อัน -> form.changeTypes = ["App", "DB"]
  ท่า 4: checkbox เดี่ยว -> ค่าเป็น true/false
     <input type="checkbox" v-model="form.downtime">

── คำที่ 4: {{ }} = โชว์ค่าตัวแปรบนหน้าเว็บ ──

  <td>{{ index + 1 }}</td>

(จาก FormView.vue — เลขลำดับแถวในตาราง)
ปีกกาคู่แปลว่า "เอาค่าตัวแปรมาแปะตรงนี้"
ค่าเปลี่ยนจอเปลี่ยนเอง แถมใส่สูตรง่ายๆ ได้ด้วย เช่น index + 1

── คำที่ 5: @click / @submit = ดักเหตุการณ์ ──

  <button @click="addRow">                กดปุ่ม -> เรียก addRow()
  <form @submit.prevent="handleSubmit">   ส่งฟอร์ม -> handleSubmit()

เจ้า .prevent ที่ห้อยท้าย = กันไม่ให้ browser reload หน้า
(ปกติ submit ฟอร์มปุ๊บ browser จะโหลดหน้าใหม่ ค่าที่กรอกหายเกลี้ยง)

ส่วน function ที่ถูกเรียก เขียนรวมกันไว้ในก้อน methods:

  methods: {
    addRow() {
      this.rows.push({ step: "", start: "", ... });
    }
  }

จุดสังเกตสำคัญ: ใน methods ถ้าจะแตะตัวแปรใน data()
ต้องมี this. นำหน้าเสมอ เช่น this.rows / this.username
(this = ตัว component นี้เอง)

── คำที่ 6: v-for = วนวาดของซ้ำๆ จากรายการ ──

  <tr v-for="(row, index) in rows" :key="index">

(จาก FormView.vue) อ่านว่า: "สร้าง <tr> 1 อัน ต่อ 1 สมาชิกใน rows"

  rows มี 1 ตัว  -> ตารางมี 1 แถว
  rows มี 5 ตัว  -> ตารางมี 5 แถว
  row   = ข้อมูลของแถวนั้น (row.step, row.owner, ...)
  index = ลำดับแถว เริ่มนับที่ 0 (เลยต้องโชว์ index + 1 ไง)
  :key  = ป้ายชื่อประจำแถว ให้ Vue จำได้ว่าแถวไหนคือแถวไหน

สังเกตไหมว่าใน FormView.vue ไม่มี <tr> ข้อมูลเขียนตายตัวเลยสักแถว
มีแต่ v-for บรรทัดเดียว — แถวทั้งหมด "เกิดจากข้อมูล" ใน rows ล้วนๆ


▶ Vue ตอนที่ 3: ตามรอยข้อมูล 1 เส้น (10 นาที เก็ท Vue ทั้งระบบ)
──────────────────────────────────────────────

เปิดหน้าฟอร์มใน browser + เปิด src/views/FormView.vue ใน VSCode
แล้วไล่ดูว่าตอนกดปุ่ม "+ เพิ่มขั้นตอนงาน" เกิดอะไรขึ้นบ้าง:

  1. เรากดปุ่ม
     <template>: <button @click="addRow">
     -> Vue วิ่งไปเรียก method ชื่อ addRow

  2. addRow ทำงาน (อยู่ในโซน <script> ไฟล์เดียวกัน)
     this.rows.push({ step: "", start: "", end: "", ... })
     -> ยัด object ว่างเข้า array rows เพิ่ม 1 ตัว
     ★ หน้าที่เราจบตรงนี้เลยนะ สังเกตว่าไม่มีโค้ด
       สั่งวาดจอเลยสักบรรทัด

  3. Vue เห็นเองว่า rows ยาวขึ้น
     -> v-for วาด <tr> เพิ่มให้ 1 แถว

  4. เลขลำดับ {{ index + 1 }} ของทุกแถวถูกคำนวณใหม่เอง

ลองกดปุ่ม "ลบ" ดูด้วย — เส้นทางเดียวกันเป๊ะ
แค่เปลี่ยนจาก push เป็น splice (ดึงสมาชิกออกจาก array)
แล้ว Vue ก็ลบแถวออกจากจอให้เอง

ถ้าให้สรุป Vue ทั้งตัวเหลือประโยคเดียว:

    "แก้ข้อมูลอย่างเดียว จอเปลี่ยนตามเอง"

เข้าใจเส้นนี้เมื่อไหร่ = เข้าใจ Vue ไปแล้ว 80%

▶ Vue ตอนที่ 5: โจทย์ฝึก Vue (ง่าย -> ยาก)
──────────────────────────────────────────────

โจทย์ V1: โชว์ค่าสดๆ
  ใน src/views/FormView.vue เพิ่ม  <p>ผู้ร้องขอ: {{ form.requester }}</p>
  ตรงไหนก็ได้ที่อยู่ในโซน <template>
  -> พิมพ์ชื่อในช่องผู้ร้องขอ แล้วดูข้อความเปลี่ยนตามสดๆ
  (v-model + {{ }} ทำงานร่วมกันให้เห็นกับตา)

โจทย์ V2: ปุ่มนับเลข
  เพิ่ม counter: 0 ใน data() ของ FormView.vue
  เพิ่มปุ่ม <button type="button" @click="counter++">กด</button>
  แล้วโชว์ด้วย {{ counter }}

โจทย์ V3: นับจำนวนขั้นตอน
  ใต้ตาราง action plan เพิ่ม
  <p>ทั้งหมด {{ rows.length }} ขั้นตอน</p>
  -> กดเพิ่ม/ลบแถวแล้วเลขต้องเปลี่ยนเอง

โจทย์ V4: คำเตือนงานด่วน (ได้คำใหม่แถม: v-if)
  v-if = โชว์ element นี้เฉพาะตอนเงื่อนไขเป็นจริง
  <p v-if="form.priority === 'Critical'" style="color: red">
    งานด่วนมาก! ต้องแจ้งหัวหน้าแผนกก่อน
  </p>
  -> ลองติ๊ก priority ไปมา ข้อความต้องโผล่/หายเอง

โจทย์ V5: ปุ่มล้างฟอร์ม
  เพิ่ม method ใน FormView.vue เซ็ตตัวแปรใน this.form กลับเป็นค่าว่าง
  + เซ็ต this.rows กลับเหลือแถวว่าง 1 แถว
  แล้วเพิ่มปุ่มในโซน <template> ผูกด้วย @click
  (เก๋ขึ้นอีกนิด: ใช้ confirm("ล้างฟอร์มทั้งหมด?") ถามก่อนล้างด้วย)

ครบ 5 ข้อ = ใช้ Vue เป็นแล้ว ระดับอ่านออก แก้ได้ เพิ่มของเองได้


▶ Vue ตอนที่ 6: กับดักที่มือใหม่โดนกันทุกคน
──────────────────────────────────────────────

  1. เพิ่มช่องกรอกใน <template> แล้วลืมเพิ่มตัวแปรใน data()
     -> เปิด Console (F12) จะเห็น Vue บ่นสีเหลือง/แดง
     กติกา: v-model ผูกกับตัวแปรไหน ตัวแปรนั้นต้องมีใน data()

  2. ลืม this. ใน methods
     เขียน rows.push(...) เฉยๆ -> พังทันที
     error ว่า "rows is not defined"
     ต้องเป็น this.rows.push(...) เสมอ

  3. แก้โค้ดแล้วจอไม่เปลี่ยน
     เช็คตามลำดับ: save หรือยัง -> เทอร์มินัลที่รัน npm run dev
     ยังไม่ error แดงใช่ไหม -> Console ใน browser มี error แดงไหม
     (error เดียวอาจทำทั้งหน้าหยุดทำงานได้เลยนะ)

  4. เขียน {{ }} หรือ v-model ไว้นอก <template>
     -> ไฟล์คอมไพล์ไม่ผ่านเลย (ไม่เหมือนเมื่อก่อนที่แค่โชว์ปีกกาดิบๆ)
     ของ Vue ทุกอย่างต้องอยู่ "ข้างใน" <template> ของไฟล์ .vue นั้นเท่านั้น

  5. เปิด tutorial ในเน็ตแล้วเจอ setup() / ref() / <script setup>
     หน้าตาไม่เหมือน views/*.vue ในโปรเจคเราเลย งงเลย
     -> นั่นคือ "Composition API" เป็น Vue อีกสไตล์ (นิยมในโปรเจคใหญ่ๆ)
     หน้า (views) ในโปรเจคเราใช้ "Options API" (แบบ data/methods)
     ซึ่งเหมาะกับมือใหม่กว่า — เรียนแบบนี้ให้แน่นก่อน
     (มีข้อยกเว้นเดียว: src/App.vue ใช้ <script setup> เพราะแค่
      โชว์ <router-view/> สั้นๆ ไม่มี logic อะไรให้งง ข้ามไปได้เลย)
     เดี๋ยวค่อยไปทำความรู้จัก Composition API อีกแบบทีหลัง

▶ Backend ตอนที่ 1: หลังบ้านคืออะไร ทำไมต้องมี
──────────────────────────────────────────────

เมื่อก่อนกด Submit แล้วข้อมูลแค่โชว์ใน Console เฉยๆ
ปิด browser = หายเกลี้ยง เพราะทุกอย่างอยู่ฝั่ง browser หมด

หลังบ้าน = โปรแกรมอีกตัวที่รันแยกจาก browser
หน้าที่: รับข้อมูล -> ตรวจ -> เก็บลง database -> ตอบกลับ

นึกภาพร้านอาหาร:
  หน้าบ้าน (frontend)  = โต๊ะอาหาร เมนู พนักงานรับออเดอร์
  หลังบ้าน (backend)   = ครัว — ลูกค้าเข้าไปเองไม่ได้
  database             = ตู้เย็น/สต็อกของ — ครัวเท่านั้นที่เปิดได้

ภาษาที่ใช้เขียนครัวของเรา: JavaScript ตัวเดิมเป๊ะ
แค่ย้ายที่รัน — เดิมรันใน browser ตอนนี้รันด้วย Node.js
ส่วน Express = ตัวช่วยให้เขียน server สั้นลงเยอะ

ศัพท์ใหม่ 4 คำ รู้แค่นี้อ่านโค้ด backend ออกหมด:

  API       = ช่องทางให้หน้าเว็บคุยกับ server
              เช่น POST /api/login = "ขอ login หน่อย"
  route     = กติกาว่า URL ไหนให้ function ไหนทำงาน
              (อยู่ใน backend/src/routes/)
  middleware = ด่านตรวจก่อนถึง route จริง
              ของเราคือด่านเช็คว่าคนยิงเป็น user ไหน (ยังไม่ login ห้ามผ่าน)
  X-User-Id = "ป้ายชื่อ" ที่หน้าเว็บแนบมากับทุก request หลัง login
              บอก server ว่าคนยิงคือ user เลขไหน
              (แบบง่ายไว้หัดเขียน — งานจริงใช้ token/session ที่ปลอมไม่ได้)


▶ Backend ตอนที่ 2: ตั้งค่าครั้งแรก (ทำครั้งเดียวจบ)
──────────────────────────────────────────────

ต้องมี SQL Server ในเครื่องก่อน (database ที่เก็บข้อมูลจริง):
  Mac:     ไม่มีตัวติดตั้งตรงๆ — ใช้ Docker แทน:
           docker run -e "ACCEPT_EULA=Y" -e "MSSQL_SA_PASSWORD=รหัสของเรา" \
             -p 1433:1433 --name sql1 -d mcr.microsoft.com/mssql/server:2022-latest
           แล้วต่อด้วย Azure Data Studio หรือ extension "mssql" ใน VSCode
  Windows: โหลด SQL Server Express/Developer จาก
           microsoft.com/sql-server/sql-server-downloads
           ตอนติดตั้งเลือกโหมด "Mixed Mode" แล้วตั้งรหัส user sa — จดไว้ให้ดี
  ทั้งคู่: ต่อดูด้วย SSMS หรือ Azure Data Studio ก็ได้ (มีปุ่ม Execute รัน .sql ตรงๆ)

จากนั้นไล่ทีละขั้น (พิมพ์ใน Terminal ที่โฟลเดอร์ IT):

  1. สร้าง database + ตารางทั้งหมดจากพิมพ์เขียว
     เปิดไฟล์ database/schema.sql ใน SSMS/Azure Data Studio แล้วกด Execute
     (หรือถ้ามี sqlcmd: sqlcmd -S localhost -U sa -P รหัส -i database/schema.sql)

  2. ติดตั้งของที่ backend ใช้:

       cd backend
       npm install

  3. สร้างไฟล์ .env (ที่เก็บรหัสลับของเครื่องเรา):

       cp .env.example .env

     แล้วเปิดไฟล์ .env แก้ 1 ที่:
       DB_PASSWORD=   ใส่รหัส user sa ของ SQL Server

  4. ตั้งรหัสผ่านให้ user ตัวอย่าง:
     รหัสผ่านห้ามเก็บดิบๆ ต้องแปลงเป็น hash ก่อน
     (hash = แปลงทางเดียว ย้อนกลับเป็นรหัสเดิมไม่ได้
      เวลา login server จะเทียบ hash กับ hash ไม่ใช่รหัสตรงๆ)

       node scripts/hash-password.js 1234

     ได้ข้อความยาวๆ ขึ้นต้น $2a$ มา ก็อปไว้ แล้วรันใน SSMS/Azure Data Studio:

       USE CR;
       UPDATE users SET password_hash = N'วางตรงนี้';

     (อัปเดตทีเดียวทั้ง 3 user — ทุกคนรหัส 1234 ไว้ฝึกพอ)

  5. สตาร์ท server:

       npm run dev

     เห็น "API server running at http://localhost:4000" = สำเร็จ
     ลองเปิด http://localhost:4000/api/health ใน browser
     เห็น {"ok":true} = server หายใจอยู่

เสร็จแล้ว! ต่อไปเปิดแค่ 2 Terminal ตามขั้นที่ 2 พอ
(backend อัน frontend อัน — ไม่ต้องตั้งค่าซ้ำอีก)

login ได้ 3 user (รหัส 1234 ทั้งหมด):
  admin      สิทธิ์ it_admin  — ทำได้ทุกอย่าง
  somchai    สิทธิ์ requester — ยื่น CR ได้ แต่อนุมัติไม่ได้
  approver1  สิทธิ์ approver  — อนุมัติได้


▶ Backend ตอนที่ 3: ตามรอย 1 เส้น — กด Sign in แล้วเกิดอะไร
──────────────────────────────────────────────

เปิด 2 ไฟล์นี้ไล่ตามไปด้วย เห็นภาพสุด:
  frontend/src/views/LoginView.vue -> backend/src/routes/auth.js

  1. เรากรอกรหัสแล้วกด Sign in
     LoginView.vue: apiFetch("/auth/login", {...})
     -> ยิงข้อมูลไปที่ http://localhost:4000/api/auth/login

  2. Express รับเรื่อง (backend/src/index.js)
     เห็น URL ขึ้นต้น /api/auth -> ส่งต่อให้ routes/auth.js

  3. auth.js ทำงาน:
     - ไปค้น username นี้ในตาราง users
     - เอารหัสที่กรอกมาเทียบกับ hash ใน DB (bcrypt.compare)
     - ผิด -> ตอบ 401 กลับไป (หน้าเว็บ alert แจ้ง)
     - ถูก -> ตอบข้อมูล user กลับไป (ชื่อ แผนก role)

  4. กลับมาฝั่ง LoginView.vue:
     เก็บข้อมูล user ลง localStorage แล้วพาไปหน้าฟอร์ม (this.$router.push)

  5. หลังจากนี้ทุกครั้งที่ยิง API
     apiFetch (ใน src/services/api.js) แนบป้ายชื่อ X-User-Id ไปให้เองอัตโนมัติ
     middleware ฝั่ง server เอาเลขไปเช็คในตาราง users ก่อนปล่อยเข้า route

เส้น Submit CR ก็ท่าเดียวกัน แค่ปลายทางคือ routes/cr.js
ที่พิเศษคือมันบันทึก 3 ตารางรวดเดียว (CR + ประเภท + แผนงาน)
ด้วยของที่เรียกว่า transaction:
  "ต้องสำเร็จครบทุกตาราง ไม่งั้นยกเลิกทั้งหมด"
  กันข้อมูลเก็บครึ่งๆ กลางๆ เวลาพังกลางทาง


▶ Backend ตอนที่ 4: โจทย์ฝึก Backend (ง่าย -> ยาก)
──────────────────────────────────────────────

โจทย์ B1: เพิ่มตัวเลือก dropdown จาก database
  INSERT INTO systems อีกแถว (ดูโจทย์ 2 ขั้นที่ 6)
  refresh หน้าฟอร์ม เห็นตัวเลือกใหม่ = เข้าใจแล้วว่า
  ข้อมูลหน้าเว็บมาจาก DB ไม่ได้เขียนตายใน template

โจทย์ B2: เพิ่ม user ใหม่ให้ตัวเอง
  สร้าง hash รหัสผ่านด้วย scripts/hash-password.js
  แล้ว INSERT INTO users เอง ลอง login ด้วยชื่อตัวเอง

โจทย์ B3: ลอง login ด้วย somchai แล้วไปกดบันทึกผลอนุมัติ
  ต้องโดน 403 เด้ง — ไปอ่าน requireRole ใน
  backend/src/middleware/auth.js ว่ามันกันยังไง

โจทย์ B4: เพิ่ม API เส้นใหม่ GET /api/me
  ตอบข้อมูล user ที่ login อยู่ (มีใน req.user อยู่แล้ว)
  ดู routes/systems.js เป็นแบบ — สั้นสุดในโปรเจค

โจทย์ B5: ทำปุ่ม "บันทึกร่าง" ให้เก็บลง DB จริง
  ตอนนี้ saveDraft ใน services/commonActions.js ยัง alert เฉยๆ
  backend รองรับอยู่แล้ว (ส่ง status: "draft" ไปกับ body)
  แก้ฝั่ง frontend ให้ยิง API เหมือน handleSubmit


▶ Backend ตอนที่ 5: พังแล้วดูตรงไหน
──────────────────────────────────────────────

ฝั่ง backend ไม่มี F12 — error โชว์ใน Terminal ที่รัน npm run dev
ไล่แบบนี้:

  "ECONNREFUSED" ตอนหน้าเว็บยิง API
    = backend ไม่ได้รันอยู่ -> ไปสตาร์ทก่อน

  "Login failed for user 'sa'"
    = รหัสใน .env ไม่ตรงกับรหัส SQL Server จริง

  "Cannot open database "CR" requested by the login"
    = ยังไม่ได้รัน schema.sql (ตอนที่ 2 ข้อ 1)

  หน้าเว็บ alert "Missing X-User-Id header"
    = ยังไม่ได้ login (หรือ localStorage โดนล้าง) -> login ใหม่

  หน้าเว็บ alert "Forbidden: insufficient role"
    = user นี้ไม่มีสิทธิ์ทำสิ่งนั้น (ดูโจทย์ B3)
