============================================================
คู่มือพัฒนาโปรเจค IT Change Request (CR)
เขียนสำหรับคนที่ "ไม่เคยเขียนโค้ดเลย" — เริ่มจาก 0 ได้จริง
อ่านทีละขั้น ห้ามข้าม ทำตามไปเรื่อยๆ เดี๋ยวเป็นเอง
============================================================


------------------------------------------------------------
ขั้นที่ 0: เตรียมเครื่อง (ครั้งแรกครั้งเดียว)
------------------------------------------------------------
ต้องมีแค่ 2 อย่าง:

1. Browser — Chrome หรือ Edge (มีอยู่แล้วทุกเครื่อง)
2. VSCode — โปรแกรมแก้โค้ด (ฟรี)
   ดาวน์โหลด: code.visualstudio.com
   ลงเสร็จ เปิด VSCode -> File -> Open Folder
   -> เลือกโฟลเดอร์โปรเจคนี้ (IT)

แค่นี้พอ ยังไม่ต้องลงอะไรอีก


------------------------------------------------------------
ขั้นที่ 1: เว็บเพจทำงานยังไง (อ่าน 2 นาที)
------------------------------------------------------------
เว็บ 1 หน้า = ไฟล์ 3 ชนิดทำงานร่วมกัน เปรียบเป็นบ้าน:

  HTML = โครงบ้าน     มีห้องอะไร ประตูอยู่ไหน
                       (ปุ่ม ช่องกรอก ตาราง อยู่ตรงไหนของหน้า)
  CSS  = สีทาบ้าน     สวยยังไง สีอะไร ขนาดเท่าไหร่
  JS   = ไฟฟ้าประปา   กดสวิตช์แล้วเกิดอะไร
                       (กดปุ่มแล้วให้ทำอะไร)

browser อ่านไฟล์พวกนี้แล้ววาดหน้าเว็บให้เราดู
โปรเจคนี้แยกไฟล์ 3 ชนิดนี้ออกจากกันชัดเจน จะได้หาง่าย


------------------------------------------------------------
ขั้นที่ 2: เปิดโปรเจคดูของจริง (5 นาที)
------------------------------------------------------------
1. เปิดโฟลเดอร์ IT ใน Finder/File Explorer
2. ดับเบิลคลิกไฟล์ index.html -> เปิดใน browser
3. หน้า login โผล่มา กรอก:
     username: admin
     password: 1234
4. เข้าหน้าฟอร์ม CR -> ลองกรอก ลองกดปุ่ม + เพิ่มขั้นตอนงาน
5. กด Submit -> ไปหน้าอนุมัติ

หมายเหตุ: ต้องต่อ internet เพราะบางส่วน (Vue, ไอคอน)
โหลดมาจากเน็ตตอนเปิดหน้า


------------------------------------------------------------
ขั้นที่ 3: แก้โค้ดครั้งแรกในชีวิต (10 นาที)
------------------------------------------------------------
ทำตามนี้เป๊ะๆ:

1. เปิด VSCode ที่โฟลเดอร์โปรเจค
2. เปิดไฟล์ index.html
3. หาบรรทัดที่เขียนว่า  <h1>Login</h1>
4. แก้เป็น             <h1>Login ระบบ CR</h1>
5. กด Save (Ctrl+S หรือ Cmd+S)
6. กลับไปที่ browser กด Refresh (F5)
7. เห็นข้อความเปลี่ยน = เขียนโค้ดเป็นแล้ว 1 บรรทัด

วงจรการพัฒนาคือแบบนี้เลย: แก้ -> save -> refresh -> ดูผล
วนแบบนี้ไปตลอดชีวิตนักพัฒนา


------------------------------------------------------------
ขั้นที่ 4: รู้จักโครงไฟล์ — อะไรอยู่ไหน
------------------------------------------------------------
  IT/
  |-- index.html      หน้า login
  |-- form.html       หน้าฟอร์ม CR
  |-- approve.html    หน้าอนุมัติ
  |-- README.md       แผนที่โปรเจคฉบับย่อ
  |-- css/            สีทาบ้าน (หน้าตา)
  |   |-- base.css    ใช้ทุกหน้า (พื้นหลัง ปุ่ม)
  |   |-- login.css   เฉพาะหน้า login
  |   |-- form.css    หน้าฟอร์ม + หน้าอนุมัติ
  |-- js/             ไฟฟ้าประปา (การทำงาน)
  |   |-- common.js   ปุ่มที่ใช้ร่วมกันทุกหน้า
  |   |-- login.js    เช็ครหัสผ่าน
  |   |-- form.js     ฟอร์ม + ตารางเพิ่ม/ลบแถว
  |   |-- approve.js  หน้าอนุมัติ
  |-- img/            รูปภาพ

กติกาของโปรเจค (จำ 3 ข้อ):
  1. HTML = โครง / CSS = หน้าตา / JS = การทำงาน ไม่เขียนปนกัน
  2. ของใช้ร่วมหลายหน้า -> base.css / common.js
  3. ของเฉพาะหน้าไหน -> ไฟล์ของหน้านั้น

ตารางลัด "อยากแก้อะไร ไปไฟล์ไหน":
  เปลี่ยนสีปุ่ม ทุกหน้า        -> css/base.css
  หน้าตาฟอร์ม/ตาราง            -> css/form.css
  เพิ่มช่องกรอกในฟอร์ม         -> form.html + js/form.js
  ข้อความปุ่มยกเลิก/ร่าง/PDF   -> js/common.js
  เปลี่ยน username/password    -> js/login.js

ทุกไฟล์มี comment ภาษาไทยอธิบายเกือบทุกบรรทัดแล้ว
(comment = ข้อความหลัง // หรือใน <!-- --> ที่ browser ไม่อ่าน
 มีไว้ให้คนอ่านเข้าใจโค้ด)


------------------------------------------------------------
ขั้นที่ 5: ศึกษาอะไร เรียงตามลำดับ (อย่าข้ามขั้น)
------------------------------------------------------------
ลำดับที่ 1 — HTML (ประมาณ 3-5 วัน)
  เรียนแค่: แท็กพื้นฐาน, form, input, table, label
  ที่เรียน: w3schools.com/html — กด "Try it Yourself"
            ลองโค้ดในเว็บได้เลยไม่ต้องลงอะไร
  หรือ YouTube ค้น "HTML พื้นฐาน" มีสอนไทยเพียบ

ลำดับที่ 2 — CSS (ประมาณ 1 สัปดาห์)
  เรียนแค่: selector, class, สี, ขนาด, flexbox, grid
  ที่เรียน: w3schools.com/css
  ฝึกกับ: เปิด css/base.css แล้วลองเปลี่ยนสีปุ่มดู

ลำดับที่ 3 — JavaScript (ประมาณ 2-3 สัปดาห์ ค่อยๆ ไป)
  เรียนแค่: ตัวแปร, if/else, function, array, object
  ที่เรียน: w3schools.com/js หรือ javascript.info
  ยังไม่ต้องเรียนหมด รู้ 5 เรื่องนี้ก็อ่านโปรเจคนี้ออก

ลำดับที่ 4 — Vue 3 (ประมาณ 1 สัปดาห์)
  โปรเจคนี้ใช้ Vue ช่วยให้ HTML กับ JS คุยกันง่าย
  เรียนแค่ 6 คำนี้พอ:
    data()      = ตัวแปรของหน้า
    v-model     = ผูกช่องกรอกกับตัวแปร (พิมพ์ปุ๊บค่าเปลี่ยนปั๊บ)
    v-for       = วนสร้างของซ้ำๆ จากรายการ (ตารางใช้ตัวนี้)
    @click      = กดปุ่มแล้วเรียก function
    methods     = function ของหน้า
    mount       = สั่ง Vue เริ่มทำงาน
  ที่เรียน: vuejs.org/tutorial (บทเรียนทางการ ลองในเว็บได้)
  วิธีที่ได้ผลสุด: เปิด js/form.js คู่กับ form.html
  ไล่อ่าน comment ไทยในไฟล์ เทียบกันทีละส่วน

ลำดับที่ 5 — เครื่องมือประจำตัว
  DevTools: กด F12 ใน browser -> แท็บ Console
    - error ขึ้นสีแดง บอกไฟล์+บรรทัดที่พัง
    - กด Submit ฟอร์มแล้วดู Console จะเห็นข้อมูลที่เก็บไว้
  Git: สมุดบันทึก version ของโค้ด ย้อนกลับได้เมื่อพัง
    เรียนคำสั่ง: git status / git add / git commit
    ที่เรียน: YouTube "git พื้นฐาน"


------------------------------------------------------------
ขั้นที่ 6: โจทย์ฝึกกับโปรเจคนี้ (เรียงง่าย -> ยาก)
------------------------------------------------------------
โจทย์ 1: เปลี่ยนสีปุ่ม Submit
  ไป css/base.css หา .btn-submit แก้ค่า background
  ลองใส่ #e91e63 ดู

โจทย์ 2: เพิ่มตัวเลือกใน dropdown "ระบบที่เกี่ยวข้อง"
  ไป form.html หา <select> เพิ่ม <option> อีกบรรทัด

โจทย์ 3: เพิ่มช่องกรอก "เบอร์ภายใน"
  ไป form.html ก็อปช่องที่มีอยู่มาแก้
  แล้วไป js/form.js เพิ่มตัวแปรใน data()

โจทย์ 4: เพิ่มปุ่ม "ล้างฟอร์ม"
  เพิ่ม method ใหม่ใน js/form.js
  แล้วเพิ่มปุ่มใน form.html ผูกด้วย @click

โจทย์ 5: เก็บร่างลง localStorage (ปิด browser แล้วข้อมูลไม่หาย)
  ค้น Google: "localStorage JavaScript วิธีใช้"

ทำครบ 5 ข้อ = เข้าใจโปรเจคทั้งตัว พร้อมต่อของจริง


------------------------------------------------------------
ขั้นที่ 7: กฎเหล็ก (ท่องไว้)
------------------------------------------------------------
1. แก้ทีละอย่าง -> save -> refresh -> ดูผล
   อย่าแก้ 10 ที่รวดเดียว พังแล้วหาไม่เจอ
2. เปิด Console (F12) ค้างไว้เสมอ แดงขึ้น = พัง อ่าน error ก่อน
3. ก่อนแก้อะไรใหญ่ๆ commit เก็บไว้ก่อน พังแล้วย้อนได้
4. โค้ดจากเน็ต/AI ต้องอ่านให้เข้าใจก่อนวาง
5. รหัสผ่านใน js/login.js เช็คฝั่ง browser
   ใช้ได้เฉพาะโปรเจคหัดเขียน งานจริงต้องเช็คฝั่ง server เท่านั้น


------------------------------------------------------------
ขั้นที่ 8: ติดปัญหาทำไง
------------------------------------------------------------
1. กด F12 อ่าน error ใน Console — มันบอกไฟล์+บรรทัดเสมอ
2. ก็อปข้อความ error ไปค้น Google ตรงๆ
3. ถาม AI (ChatGPT/Claude) แปะ error + โค้ดส่วนนั้น
4. พังหนักย้อนไม่ถูก — ถามคนให้ไฟล์นี้มา 555


------------------------------------------------------------
เส้นทางหลังจากนี้ (ไว้ค่อยดู ยังไม่ต้องสน)
------------------------------------------------------------
ตอนนี้โปรเจคเป็นแค่ "หน้าบ้าน" ข้อมูลกด Submit แล้วหายไป
ของจริงต้องมี "หลังบ้าน" รับและเก็บข้อมูล:
  Node.js + Express -> เขียน server
  SQL               -> เก็บข้อมูลลง database
  REST API          -> ให้หน้าเว็บคุยกับ server
  bcrypt + JWT      -> ระบบ login จริง
ไปถึงขั้นนั้นเมื่อไหร่ ค่อยเปิดหัวข้อนี้อ่านอีกที

จบ. เริ่มจากขั้นที่ 0 ไล่ลงมาทีละขั้น สู้ๆ
